# Google Play Requirements

An Android version would be more limited than the desktop tool. It should use Android storage permissions carefully and only access user-selected files or folders.

## Product Rules

- Use Android Storage Access Framework.
- Ask the user to select files or folders.
- Avoid broad storage permissions unless truly required.
- Avoid background storage scanning.
- Avoid destructive delete flows by default.
- Do not upload files without explicit user consent.
- Include a privacy policy.
- Complete the Data Safety form accurately.

## Metadata Needed

- App name.
- Short description.
- Full description.
- App icon.
- Feature graphic.
- Phone/tablet screenshots if supported.
- Privacy Policy URL.
- Support email.
- Category: Tools or Productivity.
- Content rating.
- Storage permission explanation.

## Realistic Android Scope

Android should be treated as a selected-storage viewer/copier, not the main advanced disk recovery edition.
